Программа принимает на вход файлы расширения *.xls -(сохранить как 2003 год ) в виде таблицы, оформленные по следующим правилам:
загружаемый файл должен быть там же где ярлык запуска программы.

Всего присуствуют 2 типа данных : числовые - такие как значения Gain, NF, и тд
			          категории - Technology, Node, Author и тд

Для корректной работы программы необходимо чтобы в таблице обязательно присуствовали столбцы:

Year - причем он самый первый(обязательно!)
Author - на любом месте, причем, кого вы считаете автором - тоже не очень важно(это может быть просто название файла статьи который хранится у вас ко компе). 
Главное, чтобы была возможность быстро найти эту статью.

Единицы измерения - Пока что оформляйте следующим образом - Gain_dB   ,  Node_nm , то есть через нижнее подчеркивание. Проценты -  PAEmax_perc

Пустые клетки
там где неизвестно значение числового параметра - оставляйте просто пустым
там где неизвестна категория (технология, техпроцесс, преминение) и др - заполняте словом - noinfo

Если у числовых параметров есть диапазон значений - например для частот - целесообразно разделить столбец на два - Fmin_GHz и Fmax_GHz

Если диапазон есть, но всего у пары компонентов в списке - то целесообразнее будет выбрать одно значение и оставить его в таблице - 
например у вас NF_dB = 3,5-4,  у остальных элементов в таблице там просто одна цифра. Выберите одно значение, отражающее суть происходящего наиболее точно.


По типам графиков:
'Scatter' - простой точечный график. X Y - любой тип данных

'Scatter Groupe' - тоже точечный, но с возможностью подсветки каждой группы - дополнительно помимо осей нужно выбрать группу - XY - !!!только числовые данные

'Pie chart'       - пирог, с секторами - показыващий в процентах долю выбранного параметра из всех данных

'Spider Plot'     - он же Radar chart  - удобно для сравнения выбранных компонентов по параметрам - выбираются !числовые! параметры и галочки напротив компонента в таблице
			не рекомендую сравнивать больше 5-6 компонентов одновременного

'GScatter'  - 	тоже точечный групповой - но XY типы данных - любые
'GScatter 3D',  - тоже что и точечный - только 3D 
'Stem',		- тоже что и точечный, но с полоской - XY тип данных любой
'GStem', 	- тоже точечный групповой - XY тип данных любой
'GStem 3D',...         - только 3D с полоской-  XY тип данных любой
'Simple Bar chart',	- выбираете комопненты - галочки напротив выбранных элементов таблицы и параметр по которому сравнивать компоненты
'Horz Bar chart',...     - тоже самое только горизонтальные столбцы
'Bar Multi-Param.', 	 - выбираете компоненты и параметры - числовые!!!
'Horz Bar Multi-Param.',... - тоже самое только горизонтальные столбцы

'Parallel Coordinates'  - выбираете параметры, группу, для ее подсветки(например технологию) и строите график парал.координат, график появляется во всплывающем окне, оси можно передвигать.
